pythonでの配列や多重配列の入力について

AtCoderを解いていてとても苦労した。
C/C++で今までコードを書いてきていたため配列の入力も従来のC/C++の書き方で問題ないかと思いきや全く違っていたいたのだ。

配列に数字を入力する場合は
C++

vector<int> v(b);
    for (int i=0; i<b; i++) {
        cin >> v[i];
    }

python

array = list(map(int, input().split()))

多重配列に数字を入力する場合は
C++

int data[a][b];
    for (int i=0; i<a; i++) {
        for (int j=0; j<b; j++) {
            cin >> data[i][j];
        }
    }

python

multi_list = [list(map(int, input().split())) for _ in range(a)]

やはりC++は静的型付けなだけありint型を宣言してから配列を作るのに対してpythonは肩を付けるという概念がないため、mapの中のintのように宣言しないとinput()の入力の際にstring型と勘違いされて入力されてしまい［i］［j］のようにリストの中身を指定する際に思いもかけないアウトプットが出てきて苦労してしまう。

ただ全体的なコードの記述量ではC++よりPythonのほうが圧倒的に少なかった。さすが可読性が高い言語と呼ばれているだけのことはあるなと感心した。

以下はコードを比較してみた結果だ。

C++

#include <iostream>
#include <vector>
#include <utility>
#include <algorithm>
#include <cstdlib>
#include <cmath>
#include <iomanip>
#include <map>
typedef long long ll;
using namespace std;
int main(void)
{
    int a, b, c;
    cin >> a >> b >> c;
    vector<int> v(b);
    for (int i=0; i<b; i++) {
        cin >> v[i];
    }
    int data[a][b];
    for (int i=0; i<a; i++) {
        for (int j=0; j<b; j++) {
            cin >> data[i][j];
        }
    }
    int cnt = 0;
    int sum = 0;
    for (int i=0; i<a; i++) {
        sum = 0;
        for (int j=0; j<b; j++) {
            sum += data[i][j] * v[j];
        }
        if (sum + c > 0)
            cnt++;
    }
    cout << cnt << endl;
    return 0;
}

Python

def main():
    a, b, c = map(int, input().split())
    array = list(map(int, input().split()))
    # 多重リスト
    multi_list = [list(map(int, input().split())) for _ in range(a)]
    cnt = 0
    for i in range(a):
        sum = 0
        for j in range(b):
            sum += multi_list[i][j] * array[j]
        if sum + c > 0:
            cnt += 1
    print(cnt)
if __name__ == "__main__":
    main()
