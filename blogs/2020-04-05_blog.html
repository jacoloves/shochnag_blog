lsをハックしてみたい

コロナウィルス影響で外出できないため、恐ろしいほど暇なのでlsコマンドをでもいじって見るかと思いcoreutilsを落としてきた。
coreutils
src/直下のlsコマンドを発見したためmakeしようとしたがconfig.hがないと言われてコンパイルできなかった。

解決方法

configureを実行しないといけないらしい。
ここに書き方が書いてあった
どうもC/C++で書かれたソフトウェアをソースコードからコンパイルしてローカルにインストールするときに使うらしい。
色々と苦労したが下記のやり方で無事コンパイルできた。

./configure FORCE_UNSAFE_CONFIGURE=1
make

デバッグ

gdbを使用してデバッグするのがこれが中々に難しく苦労するのでgdbのTUI modeを使用してソースコード見ながらデバッグをする。
詳細はこちら

※C-x C-aの繰り返しで、TUI/CUIを切り替えます

とのことなので使わせてもらう。
Ctrl-xを押したあとに1か2を押すとTUIモードに移行するらしい。
それでもなかなかソースコードを見るのはしんどいんだけど^^;
でも少しずつ読み解けるように頑張っていこう。
最近C言語しか触ってないなぁ。

コンパイルが難しかったのでFreeBSD版を使うことに

再コンパイルにやたら時間がかかったのでvagrant+FreeBSDで再挑戦してみようと思う。
公式のサイトに導入方法が書いてあったのでそれを用いることにした。
サイト

The configured shell (config.ssh.shell) is invalid and unable
to properly execute commands. The most common cause for this is
using a shell that is unavailable on the system. Please verify
you're using the full path to the shell and that the shell is
executable by the SSH user.

なんかエラーみたいなのが出てるけど、vagrant sshも通ったからOK

vagrant@freebsd:~/dev-room % echo "Hello world!"
Hello world!

新しい開発環境ができた。
